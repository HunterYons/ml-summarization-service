# Лабораторная работа №4: Развертывание и мониторинг (MLOps)

### 1. Цель работы
Целью данной работы является внедрение практик MLOps (Machine Learning Operations) для обеспечения жизненного цикла модели: автоматизация развертывания в изолированных средах, мониторинг качества генерации и управление метаданными экспериментов.

### 2. Контейнеризация и оркестрация
Для обеспечения воспроизводимости среды и независимости от системных зависимостей хост-машины была выбрана платформа **Docker**. 

Система развернута с помощью **Docker Compose** и состоит из четырех взаимодействующих сервисов:
* **db:** Реляционная база данных PostgreSQL для хранения истории запусков и артефактов MLflow.
* **mlflow:** Сервер мониторинга, обеспечивающий UI для анализа метрик.
* **api:** Бекенд-сервис на FastAPI, выполняющий тяжелые вычисления (инференс T5 и SBERT).
* **bot:** Фронтенд-интерфейс на базе Aiogram 3.x для взаимодействия с конечными пользователями.

Использование единой сети (Docker Network) позволяет сервисам обращаться друг к другу по именам (например, `http://api:8000`), что упрощает конфигурацию.

### 3. Мониторинг экспериментов (MLflow)
В проект интегрирован трекинг каждого сеанса суммаризации. В MLflow автоматически логируются следующие параметры:

1.  **Параметры инференса:** `model_name`, `max_length`, `num_beams`.
2.  **Метрики качества:** `semantic_similarity` (на базе SBERT).
3.  **Технические метрики:** `latency_sec` (время генерации), `input_len` и `output_len`.

Это позволяет отслеживать производительность системы под нагрузкой и выявлять случаи деградации качества (например, если семантическое сходство падает ниже 0.5 на определенных типах текстов).

### 4. Реализация пользовательского интерфейса
Интерфейс реализован в виде асинхронного **Telegram-бота**. 
* **Асинхронность:** Бот не блокируется во время ожидания ответа от API благодаря использованию библиотеки `httpx` и асинхронных методов Aiogram.
* **Обработка ошибок:** Внедрена система уведомлений пользователя о статусе запроса (загрузка, обработка, ошибка сервера или слишком короткий текст).
* **Безопасность:** Конфиденциальные данные (токены бота, пароли БД) вынесены в переменные окружения (`.env`), что соответствует принципам 12-Factor App.

### 5. Результаты внедрения MLOps-пайплайна
В результате выполнения работы система приобрела свойства полноценного программного продукта:
* **Автономность:** Развертывание всей инфраструктуры происходит одной командой `docker-compose up --build`.
* **Наблюдаемость:** Все действия пользователей и ответы модели прозрачны и доступны для анализа в дашборде MLflow.
* **Масштабируемость:** Архитектура позволяет легко заменить модель `T5` на более мощную или добавить дополнительные экземпляры API при росте нагрузки.

### 6. Заключение по всему циклу работ
В ходе выполнения четырех лабораторных работ был пройден путь от постановки задачи и выбора датасета до создания распределенной микросервисной системы. Использование современных архитектур (Transformers), метрик (SBERT) и инструментов эксплуатации (Docker, MLflow) позволило создать эффективный сервис автоматической суммаризации текстов на русском языке.
