# Отчет по Лабораторной работе №3: Реализация Baseline

### 1. Выбор модели
В ходе работы была выбрана модель `IlyaGusev/rut5_base_sum_gazeta` (архитектура T5). Она была предпочтена модели mBART из-за меньшего объема (977 МБ против 3.5 ГБ) и более стабильной работы в окружении Codespaces.

### 2. Анализ ошибок и итерации
* **Итерация 1:** Использование mBART приводило к генерации некорректных символов (языковой шум).
* **Итерация 2:** Попытка использования `pipeline` выявила отсутствие регистрации задачи `summarization` в текущем окружении.
* **Итерация 3 (финальная):** Реализована прямая загрузка модели и токенизатора T5, что обеспечило корректную работу на русском языке.

### 3. Результат
Модель успешно выполняет инференс на CPU, время обработки тестового фрагмента составляет менее 10 секунд.

### 4. Оценка качества и выбор метрик
Для оценки суммаризации были выбраны две ключевые метрики:
1. **ROUGE-L:** Оценка лексического сходства. Показала результат **0.32** на тестовой выборке. Занижает оценку при использовании синонимов.
2. **BERTScore (F1):** Основная метрика семантического сходства. Результат: **0.86**. 

**Вывод по метрикам:** Высокий показатель BERTScore (0.86) подтверждает, что модель успешно передает смысл оригинала, даже если итоговый текст лексически отличается от эталона. Это критически важно для борьбы с проблемой парафраза, которую не учитывает стандартный ROUGE.
