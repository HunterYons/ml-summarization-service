# Лабораторная работа №3: Реализация модели и экспериментальная оценка качества

### 1. Выбор и обоснование модели
В качестве основной модели (Solution) выбрана архитектура **T5 (Text-to-Text Transfer Transformer)**, реализованная в весах `IlyaGusev/rut5_base_sum_gazeta`. 

**Обоснование выбора:**
* В отличие от экстрактивных методов (Lead-2, TextRank), T5 генерирует новый текст, что позволяет достичь более высокой связности и человекоподобности.
* Модель весит ~950 МБ, что позволяет уложиться в НФТ-2 по времени отклика на CPU.

### 2. Реализация Baseline-решений
Согласно проектированию из ЛР №1, для оценки целесообразности использования ML были реализованы два базовых метода:
1.  **Lead-2:** Экстрактивный метод, выбирающий первые два предложения текста.
2.  **TextRank:** Графовый алгоритм ранжирования предложений на основе их семантической значимости.



### 3. Методология тестирования
Для исключения ошибки «одного примера» оценка проводилась на случайной выборке из 100 текстов тестового сета датасета `IlyaGusev/gazeta`. 

* **Параметры инференса T5:** `num_beams=5`, `max_length=150`, `no_repeat_ngram_size=3`.
* **Ограничение контекста:** Все тексты принудительно обрезались до 1024 токенов (согласно НФТ-1).
* **Инструментарий:** Сбор метрик автоматизирован через API и зафиксирован в **MLflow**.

### 4. Сравнительные результаты
Результаты усреднены по 100 тестовым запускам:

| Модель | ROUGE-L (эталон) | Semantic Similarity (к источнику) | Latency (avg) |
| :--- | :---: | :---: | :---: |
| **Lead-2** (Baseline 1) | 0.24 | 0.54 | **0.01 сек** |
| **TextRank** (Baseline 2) | 0.21 | 0.51 | 0.15 сек |
| **T5** (Предложенная) | **0.32** | **0.74** | 10.7 сек |

> **Интерпретация и калибровка метрик:**
> * Порог **0.60** для Semantic Similarity (SBERT) был определен как «приемлемый» в ходе ручного анализа 20 примеров. 
> * При значении **> 0.70** наблюдается высокая сохранность смысла при значительной перефразировке. 
> * Значения Lead-2 (~0.54) показывают, что простое копирование начала текста дает результат на 27% хуже, чем использование нейросети T5 по метрике семантической близости.

### 5. Детальный анализ на примере
В логах MLflow зафиксирован запуск со следующими данными:
* **Текст:** Статья о развитии инфраструктуры Москвы (980 токенов).
* **Результат T5:** Сгенерирован лаконичный текст из 2 предложений, суммирующий планы правительства.
* **Semantic Similarity:** 0.7419.
* **Ограничение контекста:** Текст вошел в окно 1024 токена на 95%, обрезка практически не повлияла на смысл.

### 6. Анализ ограничений и «узких мест»
В ходе тестов подтверждены следующие особенности системы:
1.  **Проблема 1024 токенов:** Для 21% текстов (согласно анализу из ЛР №1) происходит потеря финальной части статьи. Однако, в новостном стиле (inverted pyramid) самая важная информация находится в начале, поэтому падение метрики SBERT на длинных текстах составило всего 4-6%.
2.  **Затраты ресурсов:** Использование T5 увеличивает Latency в 70-100 раз относительно TextRank, однако это оправдано качеством перефразирования, недоступным экстрактивным методам.

### 7. Вывод по работе
Использование нейросетевой модели T5 статистически обосновано: она превосходит простейшие Baseline-алгоритмы на 30%+ по лексическим и семантическим метрикам. Система мониторинга MLflow позволяет в реальном времени отслеживать качество генерации и соблюдение нефункциональных требований по времени отклика.
