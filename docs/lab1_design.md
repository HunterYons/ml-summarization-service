# Лабораторная работа №1: Определение задачи и анализ данных

### 1. Предметная область
Область исследования — обработка естественного языка (NLP), а именно задача **Abstractive Summarization** (абстрактивная суммаризация). В отличие от экстрактивной суммаризации, которая просто копирует важные предложения, выбранный подход позволяет генерировать новый текст, перефразируя основное содержание оригинала.

### 2. Цели и задачи проекта
* **Бизнес-цель:** Оптимизация процесса потребления информации пользователем. Сокращение времени на ознакомление с длинными текстами (новости, статьи, отчеты) при сохранении ключевого смысла.
* **Техническая задача:** Разработка отказоустойчивого микросервиса с API-интерфейсом и Telegram-ботом, обеспечивающего инференс нейросетевой модели-трансформера в контейнеризированной среде.
* **Задача мониторинга:** Внедрение системы трекинга экспериментов для анализа качества генерации и времени ответа модели в реальном времени.

### 3. Исследуемые данные
В качестве эталонного набора данных для тестирования и оценки качества используется датасет **IlyaGusev/gazeta** (корпус новостей Gazeta.ru). 
* **Входные данные:** Тексты новостных статей на русском языке (до 1024 токенов).
* **Целевая переменная:** Краткая аннотация (summary), написанная профессиональными редакторами.

### 4. Выбранные метрики качества
Для комплексной оценки работы модели выбраны две дополняющие друг друга метрики:
1.  **ROUGE-L:** Оценка лексического сходства на основе поиска самой длинной общей подпоследовательности (LCS). Позволяет оценить, насколько слова в суммаризации совпадают с оригиналом.
2.  **Semantic Similarity (SBERT):** Косинусное сходство между векторными представлениями (эмбеддингами) оригинала и результата. Это ключевая метрика, так как она позволяет оценить сохранение **смысла** текста, даже если модель использовала синонимы или полную перефразировку.
3.  **Inference Latency:** Время генерации ответа (целевой показатель: < 15 сек на CPU).

### 5. Контекстная диаграмма системы
Система спроектирована по микросервисной архитектуре и включает следующие узлы взаимодействия:

[Пользователь] 
      │
      ▼
[Telegram Bot (Aiogram 3)] <───> [Summarization API (FastAPI)]
      │                                 │
      │                                 ▼
      │                      [ML Model (T5 Transformer)]
      │                                 │
      └─────────────────────────────────┼───────> [MLflow Tracking]
                                        │               │
                                        ▼               ▼
                                 [PostgreSQL] <─── [Artifact Storage]

### 6. Технологический стек
* **Модель:** `IlyaGusev/rut5_base_sum_gazeta` (T5-base).
* **Метрики:** `SentenceTransformer` (модель `paraphrase-multilingual-MiniLM-L12-v2`).
* **Инфраструктура:** Docker, Docker Compose, MLflow, PostgreSQL.
