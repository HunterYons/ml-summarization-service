# Лабораторная работа №2: Архитектура и стратегия валидации

### 1. Стратегия валидации
* **Метод:** Hold-out validation (80/10/10).
* **Предотвращение утечек:** Удаление дубликатов, очистка метаданных (даты, авторы), которые могут присутствовать в тексте статьи.

### 2. Масштабируемость и производительность
* **Truncation:** Ограничение входной последовательности до 1024 токенов для стабильного потребления RAM.
* **Async IO:** Использование FastAPI для асинхронной обработки запросов, чтобы избежать блокировки сервера при ожидании инференса.

### 3. Технологический стек
* **Язык:** Python 3.10+
* **Framework:** FastAPI
* **ML:** Hugging Face Transformers (модель mbart_ru_sum_gazeta)
* **Инфраструктура:** Docker
