# ML Text Summarization Service

Интеллектуальный микросервис для автоматической суммаризации (сжатия) текстов на русском языке. Система построена на базе современных нейросетевых архитектур (Transformer) и включает в себя полноценный цикл мониторинга качества модели.

## Основные возможности

* **Абстрактивная суммаризация:** Генерация кратких выжимки с помощью модели `T5`, которая не просто вырезает предложения, а перефразирует их.
* **Умная оценка качества:** Внедрена метрика **Semantic Similarity** на базе `SBERT` для оценки сохранения смысла (вместо простого сравнения слов).
* **Telegram Bot интерфейс:** Удобное взаимодействие с пользователем в реальном времени.
* **MLflow Интеграция:** Полный трекинг каждого запроса, включая логирование точности, времени обработки и параметров модели.
* **Оптимизация (Batching):** Поддержка эффективной обработки запросов в многопоточном режиме.
* **Полная контейнеризация:** Развертывание всей инфраструктуры одной командой через Docker Compose.

## Архитектура системы

Проект разделен на независимые функциональные блоки (микросервисы):

1.  **Summarization API (FastAPI):** Сердце системы. Выполняет инференс нейросетей и расчет метрик.
2.  **Telegram Bot (Aiogram 3):** Асинхронный клиент для взаимодействия с пользователями.
3.  **MLflow Server:** Платформа для мониторинга ML-экспериментов.
4.  **PostgreSQL:** База данных для хранения метаданных MLflow и истории запросов.

## Используемые модели

* **Основная модель:** `IlyaGusev/rut5_base_sum_gazeta` (архитектура T5). Специализируется на суммаризации русскоязычного новостного контента.
* **Модель метрик:** `paraphrase-multilingual-MiniLM-L12-v2` (архитектура SBERT). Используется для создания семантических векторов текста и расчета косинусного сходства.

## Мониторинг и метрики (MLflow)

В проекте реализован продвинутый подход к оценке качества. Для каждого запроса в MLflow фиксируются:
* `semantic_similarity`: Косинусное сходство между эмбеддингами оригинала и результата (от 0 до 1).
* `latency_sec`: Время, затраченное на генерацию текста.
* `input_len` / `output_len`: Коэффициент сжатия текста.

> **Зачем это нужно?** Традиционные метрики (типа ROUGE) плохо оценивают перефразирование. Использование SBERT позволяет понять, сохранила ли модель **суть** сообщения, даже если она использовала другие слова.

## Быстрый старт

### Предварительные требования
* Установленный **Docker** и **Docker Compose**.
* Токен Telegram бота (получить у [@BotFather](https://t.me/BotFather)).

### Развертывание

1. Клонируйте репозиторий:
   ```bash
   git clone [https://github.com/ваш-аккаунт/ml-summarization-service.git](https://github.com/ваш-аккаунт/ml-summarization-service.git)
   cd ml-summarization-service

2. Создайте файл .env в корне проекта и добавьте ваши данные:

    TG_TOKEN=ваш_токен_телеграм_бота
    API_URL=http://api:8000/summarize
    DATABASE_URL=postgresql://postgres:postgres@db:5432/mlflow
    MLFLOW_TRACKING_URI=http://mlflow:5000

3. Запустите проект:

    docker-compose up --build
    Дождитесь загрузки моделей (в логах api-1 появится сообщение: Application startup complete).

### Использование
    Telegram Bot: Найдите вашего бота в Telegram и отправьте любой текст длиннее 10 символов.

    Swagger API: Доступен по адресу http://localhost:8000/docs.

    MLflow UI: Просмотр графиков и метрик по адресу http://localhost:5000.
