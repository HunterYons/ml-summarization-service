# ml-summarization-service
Лабораторная работа №1: 
Проектирование сервиса суммаризации

1. Предметная область и задача
Предметная область: Обработка естественного языка (NLP) и автоматизация работы с текстовым контентом.
Конкретная задача ML: Автоматическая генерация краткого содержания (суммаризация) текста с веб-страниц для экономии времени пользователя.

2. Цели и источники данных
Цель: Разработать дизайн и прототип сервиса, который принимает URL-адрес или текст статьи и возвращает сжатый пересказ основных мыслей.
Источник данных: В качестве обучающего и тестового набора данных будет использован открытый датасет Gazeta.ru или CNN/Daily Mail из библиотеки Hugging Face (они содержат пары «полный текст — краткое содержание»).

3. Метрики
Для оценки качества работы сервиса выбраны следующие метрики:
ROUGE (Recall-Oriented Understudy for Gisting Evaluation): Основная ML-метрика для сравнения сгенерированного текста с эталонным.
Latency (Задержка): Время обработки одного запроса (критично для пользовательского сервиса).
Длина результата: Отношение объема суммаризации к объему исходного текста.

4. Высокоуровневая архитектура
Client (Web/CLI): Отправляет URL или текст.
FastAPI Application: Принимает запрос, очищает текст от HTML-тегов (Preprocessing).Inference Module: Предобученная модель (например, IlyaGusev/mbart_ru_sum_gazeta для русского языка).
Docker Container: Весь сервис упакован в контейнер для воспроизводимости.
